### 4.2 Codex 首次复审结论：Rejected

401 项绿灯不构成切片 3 通过。Codex 对 `ddf1b6c..e49b590` 的生产代码和第 10 节矩阵逐项核对后确认下列阻塞项：

1. **流式枚举不能完成大目录。** `FolderDirectoryEnumerator` 在第 `workUnitLimit + 1` 项直接返回 `finished=false`；生产默认值 256，因此任何超过 256 个枚举项的目录都会在首批后停止并被判 incomplete，而不是提交安全边界后继续同一 enumerator 至耗尽。现有 `testStreamingBoundaryDoesNotExceedInjectedLimit` 反而固化了错误行为。
2. **Job/Application seam 仍依赖具体实现。** handler 通过向下转型 `GRDBFolderReconcileRepository` 读取 Job，`GRDBJobLeaseContextProvider` 又向下转型 `GRDBJobQueue` 并可 `fatalError`。必须由不含 GRDB 类型的端口提供执行上下文并结构化拒绝错误装配；生产路径不得靠 concrete cast 或进程崩溃维持不变量。
3. **契约校验不严格。** Payload 的 UUID 比较接受大写；测试只覆盖一个额外字段。缺少 payload/checkpoint 的缺字段、错误类型、版本、非法/大写 UUID、source 不一致、负数、attempt 与当前 claim 不一致、结构化列与 checkpoint 不一致，以及“目录访问前拒绝”的完整正反矩阵。
4. **安全错误结算错误。** `stopIncomplete` 无条件提交 `.retryableFailure`，导致 `folderPayloadInvalid`、`folderCheckpointInvalid`、`folderAuthorizationRequired`、`folderUnsafeRelativePath` 不能按合同进入 non-retryable；unsafe 分支返回值也写成 retryable。必须由明确 outcome 驱动持久化，并证明 `last_error_message = NULL`。
5. **未继承切片 2 的授权/状态语义。** 新 root adapter 绕开已批准的 stale bookmark 刷新、root validation、offline/authorizationRequired 分类与 Source 状态更新；还没有 root 中途失效和所有异常路径的 scope start/stop 配对证据。
6. **批次和 final 缺少持久化边界验证。** Repository 未在业务写前验证 Job kind/version/payload/source、输入 generation/started epoch/checkpoint/attempt 与 Job/Source 一致，也未完整验证 observation、路径/file name、批次上限。`completeGeneration` 可用任意 generation 标 missing；final 遇 pause/cancel 只是抛错回滚，未按安全边界离开 running。
7. **跨 attempt progress 语义错误。** checkpoint 计数按 attempt 归零，但批次直接把 `candidate_files` 写成 `progress_completed`；部分失败后的新 attempt 若早期候选更少，会触发 progress 回退，而不是维持跨 attempt 高水位。
8. **身份与移动规则未完成。** identity conflict 从未反馈到 checkpoint；resource ID 多候选/探测失败不累计冲突。移动重连查询到旧路径后直接忽略，未在当前 security scope 内探测旧 locator；因此旧路径仍存在的硬链接也可能被错误搬迁。relocate 以伪造的 `size=0/mtime=0` 计算 revision，几乎总会错误加一。
9. **媒体分类和 metadata 不符合合同。** 损坏 PNG 当前可能被归为 unsupported；测试为过绿灯而从“必须 unreadable”弱化成“只要不是 available”。六类允许格式只有 PNG 正例，缺少 JPEG/HEIC/HEIF/TIFF/WebP、GIF/排除格式、PDF/视频、合法容器伪扩展、动画/多帧、orientation 1～8、带/不带 offset 时间、resource ID 非路径构造及数据库 SHA NULL 证据。
10. **事务 fault 矩阵名存实亡。** test support 创建了 batch/final 多个 trigger，但测试只实际启用了 begin fault。缺少 Asset/fingerprint/checkpoint-progress-lease 后段回滚、final missing/completion/successor 三个后段回滚、retry/reopen generation 复用、无 dirty successor、pause/cancel 优先、stale owner/attempt、恰好到期、续租和 recovery 幂等。
11. **枚举/路径矩阵严重缺口。** 缺 hidden、大小写 `.photoslibrary`、alias、socket/FIFO、硬链接、多层目录、Unicode 不正规化、空/`.`/NUL/严格根边界、目录 error handler、root 中途失效与“不预收集完整目录”的证据；资源值读取失败当前被普通 ignored 吞掉。
12. **隐私/只读回归证据不足且存在误导性测试。** Privacy 测试只读源码 manifest，不验证 built App；fixture 快照只比较文件字节，不比较目录结构、mtime 和关键 metadata；migration 测试被弱化为只检查已知 ID 列表，却仍命名为“files unchanged”。真实零 diff 由 Codex 另行确认，但测试不得作出未证明的声明。
