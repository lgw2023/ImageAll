> 状态：Approved — `UI-001` 至 `UI-011` 与三项产品基线已由项目所有者批准<br>
> 日期：2026-07-15<br>
> 基线：`main@d38893ee359bb829336ca75fdf2dc8551b0a7efa`<br>
> 批准记录：项目所有者于 2026-07-15 明确批准全部决策；实施仍以单独 Cursor 交接单为准<br>

## 1. 本轮结论

项目所有者提供的前端材料确立了长期产品方向：

> Apple 原生应用外壳 + Eagle 式素材工作台 + Lightroom 式连续审核效率 +
> Aftershoot 式人机协作 + Linear/Raycast 式快捷操作。

阶段 1 不直接实现这套长期形态的全部工作区。它只建立以后可以自然扩展的产品骨架，并完成第一个真实用户闭环：

```text
连接只读文件夹
→ 立即进入统一图库
→ 在后台索引过程中逐步看到照片
→ 浏览、选择和查看基本信息
→ 创建并批量应用人工标签
→ 搜索与筛选
→ 处理离线、权限和单文件错误
```

用户材料是长期设计方向和参考库；阶段 1 只实现本文明确列出的最小闭环。项目所有者已经批准下文稳定编号的决策，后续实施不得把参考材料中的阶段外能力静默带入阶段 1。任何代码变更仍须经过独立 Cursor 交接单；任何真实照片验证仍须经过专门授权门。

## 2. 产品原则

### 2.1 照片优先

中央区域以照片为主角。工具、标签和元数据围绕当前选择出现，不用高饱和品牌色、厚边框或大量常驻文字与图片竞争注意力。

### 2.2 原生且高吞吐

采用 macOS 标准 Sidebar、Toolbar、Inspector、菜单、上下文菜单和选择语义。高频操作必须支持批量选择和键盘路径，但阶段 1 不提前设计一套与真实功能不相符的庞大快捷键系统。

### 2.3 人工事实清晰

阶段 1 只有人工决定，不显示 AI 置信度或伪造建议状态。界面必须区分：

- 已确认属于某标签；
- 已明确拒绝某标签；
- 尚未判断。

清除一个已有决定表示回到“尚未判断”，不能把“移除已接受”静默解释为“明确拒绝”。

### 2.4 渐进披露

默认界面只保留浏览与整理需要的控件。高级元数据、错误详情和批量任务细节按需展开；未来的高级布尔查询、自动化规则和模型参数不占用阶段 1 主界面。

### 2.5 对大资料库诚实

索引、缩略图生成、来源离线和部分错误是产品的一等状态。界面不能用一个无限旋转的小图标掩盖长任务，也不能在总量未知时展示虚假的完成百分比。

## 3. 阶段边界

| 能力 | 阶段 1 | 后续阶段 |
|---|---|---|
| 文件夹来源 | 连接、停用、重新授权、重扫、离线状态 | 来源规则与更高级管理 |
| Apple Photos | 不显示不可用占位入口 | 阶段 2 通过 PhotoKit 接入同一图库 |
| 图片浏览 | 统一网格、单图查看、基本元数据 | Compare、Survey、相似组 |
| 标签 | 创建、确认、拒绝、清除决定、批量操作 | AI 建议、置信强度与审核队列 |
| 筛选 | 搜索、来源、标签和系统状态 | Smart Collection、完整布尔构建器 |
| 后台任务 | 活动中心、聚合进度、安全控制和错误摘要 | 资源限制、执行历史和高级调度 |
| 命令入口 | 只执行当前已存在动作的轻量命令面板 | 自然语言查询、自动化与插件动作 |
| 自动化规则 | 不实现 | Dry Run、冲突分析和可撤销规则 |

阶段 1 不展示“即将推出”的 Apple Photos、Review、Compare、Smart Sets 或 AI 面板。产品导航只承诺当前可用能力，避免空入口和误导状态。

## 4. 主窗口信息架构

### UI-001：统一图库作为默认入口 — Approved

只要目录库已有至少一个来源，应用启动后默认打开“全部照片”，而不是某个文件夹。统一图库是长期主模型；阶段 2 的 Photos 资产以后加入同一网格，不另建一套浏览产品。

左侧 Sidebar 的阶段 1 顶层结构为：

```text
图库
  全部照片
  无标签

来源
  <用户连接的文件夹来源>
  + 连接文件夹…

标签
  <用户标签>
  + 新建标签…

活动
  后台任务与问题数量
```

不在 Sidebar 展开整个磁盘目录树。来源是稳定顶层导航；相对目录作为本地可搜索/可筛选的元数据，并在 Inspector 中显示。这样避免几个 TB 的目录层级成为第二套事实系统，也避免启动时加载大树。以后若实际使用证明用户强依赖目录导航，再增加按需展开的虚拟目录视图。

### UI-002：稳定三栏工作台 — Approved

窗口采用可折叠三栏：

```text
┌─────────────────────────────────────────────────────────────────────┐
│ Toolbar: 导航 | 搜索与筛选 | 缩略图大小 | 命令 | 活动 | Inspector │
├──────────────┬───────────────────────────────────┬──────────────────┤
│ Sidebar      │ Content                           │ Inspector        │
│              │                                   │                  │
│ Library      │ Photo Grid / Single Photo         │ Preview          │
│ Sources      │                                   │ Manual Tags      │
│ Tags         │                                   │ File Metadata    │
│ Activity     │                                   │ Source / Status  │
├──────────────┴───────────────────────────────────┴──────────────────┤
│ Context status: 选择数 / 索引状态 / 非阻塞错误摘要                 │
└─────────────────────────────────────────────────────────────────────┘
```

- Sidebar 与 Inspector 可分别折叠；
- 窗口变窄时先隐藏 Inspector，不压缩网格到不可用；
- Content 只在网格与单图查看之间切换，阶段 1 不提供瀑布流、列表、Compare 或 Survey；
- 选择和筛选状态在折叠 Inspector 后仍能从 Toolbar 或状态栏理解。

## 5. 首次使用与来源管理

### UI-003：先解释只读，再由用户连接 — Approved

空目录库显示单页空状态，而不是自动弹出系统选择器：

```text
ImageAll 在原位置读取照片

不会导入、移动、重命名或删除原图。
索引、标签和缩略图保存在 ImageAll 自己的应用容器中。

[连接照片文件夹…]
```

用户点击后才打开系统目录选择器。产品统一使用“连接文件夹”，不用容易暗示复制的“导入”。

选择成功后：

1. 当前窗口立即进入该来源的图库；
2. 后台开始对账，不用阻塞式“等待导入完成”页面；
3. 已完成的缩略图逐步出现在稳定位置；
4. Toolbar/状态栏显示真实活动状态；
5. 总量未知时显示“已扫描 N 项”和当前阶段，不显示百分比；
6. 只有总工作量可稳定计算后才显示确定型进度。

来源行的上下文菜单提供：

- 在图库中查看；
- 立即重扫；
- 重新授权；
- 暂停/继续相关任务；
- 停用来源；
- 查看来源信息。

阶段 1 不提供“删除来源并删除全部目录事实”的入口。停用来源保留资产、标签和历史。

## 6. 照片网格与单图查看

### UI-004：方形密集网格，详情保留完整比例 — Approved

默认网格使用统一方形单元和视觉居中的 `aspect fill` 展示；裁切只发生在可重建的网格派生图，不生成或修改裁切后的原图。选择单图后，Content 或 Inspector 使用中性背景 `aspect fit` 展示完整画面。

选择方形网格而不是瀑布流的原因：

- 相同行列让快速扫视和范围选择更稳定；
- 大资料库分页与滚动位置更容易预测；
- 混合横竖图不会制造剧烈布局跳动；
- 后续审核队列可复用同一选择模型。

Toolbar 提供连续或分档的缩略图尺寸调节，但 UI 尺寸不等于为每个尺寸生成独立缓存。后端建议只生成两种视觉居中的方形网格派生图，并按需生成一种保留比例的预览派生图：

| 派生用途 | 建议尺寸 | 阶段 1 用途 |
|---|---:|---|
| Grid small | 256 × 256 px | 高密度网格 |
| Grid regular | 512 × 512 px | 默认与大缩略图网格 |
| Preview | 最大边 2048 px，保留比例 | 按需用于单图查看与 Inspector 预览 |

这些数值随 UI-004 一并批准为阶段 1 缓存契约；Retina 显示、滚动内存和缓存占用仍须在相应实施切片用合成 fixture 实测，性能不达门时必须回到架构评审，不能由实现者静默改变 variant。

### 6.1 网格单元信息

默认常驻信息保持最少：

- 图片本身；
- macOS 标准选择高亮；
- 不可用/不可读/不支持的克制状态徽标；
- 已有人工作出决定时的轻量标签数量提示。

文件名、日期、来源、相对路径和标签列表不在每个单元常驻铺开。它们出现在 Hover 辅助信息、Inspector 或筛选结果摘要中。阶段 1 不在缩略图上显示 AI 色彩、置信百分比、评分、旗标或模型版本。

### 6.2 浏览行为

- 单击选择；
- `Command` 单击增减不连续选择；
- `Shift` 单击做稳定排序下的范围选择；
- 方向键移动主选择；
- `Space` 切换单图快速查看；
- `Escape` 返回网格或关闭临时界面；
- 滚动、分页加载和缩略图失败不得清除当前选择；
- 当前选择跨筛选后不可见时，明确清除并给出非打断提示，不能对隐藏项继续执行批量操作。

## 7. Inspector 与人工标签

### UI-005：Inspector 是标签主工作面 — Approved

单选 Inspector：

1. 较大预览；
2. 人工标签及三态决定；
3. 媒体日期、像素尺寸、格式和文件大小；
4. 来源、相对位置和可用状态；
5. 按需展开的技术错误详情。

多选 Inspector：

1. 选中数量；
2. 所有选中项共有的标签；
3. 部分选中项具有的标签，以 mixed 状态呈现；
4. 给全部选中项确认标签、拒绝标签或清除决定；
5. 执行前明确显示影响数量，执行后支持一次 Undo。

创建标签直接在 Inspector 标签输入框完成。不存在的有效名称显示“创建标签‘…’并应用”，不用离开当前照片进入设置页。

标签的三态交互定义为：

| UI 状态 | 领域语义 | 操作结果 |
|---|---|---|
| `✓` | `manualAccepted` | 明确认可该标签 |
| `×` | `manualRejected` | 明确否认该标签 |
| `—` | 无持久化决定 | 清除既有人工决定 |

阶段 1 不把普通“删除 chip”直接映射成拒绝；必须通过明确动作表达拒绝。

### UI-006：轻量命令面板 — Approved

阶段 1 提供 `Command-K` 命令面板，但只聚合已经存在且有权限执行的动作：

- 跳转“全部照片”、来源、标签或活动；
- 搜索并应用/拒绝/清除标签决定；
- 新建标签；
- 连接文件夹；
- 重扫当前来源；
- 切换 Sidebar、Inspector 和单图查看；
- 显示快捷键。

它不是自然语言代理，不解析“低于 60%”等尚不存在的模型条件，也不提供自动化、导出、相似搜索或阶段外入口。上下文菜单和菜单栏同时显示已实现动作的快捷键，逐步教育用户，而不是要求用户先学习快捷键表。

数字键映射常用标签、拖放标签和可编辑快捷键配置留到实际工作流验证后决定，不进入阶段 1。

## 8. 搜索、筛选与排序

### UI-007：简单筛选优先，明确 ALL/ANY — Approved

阶段 1 搜索范围：

- 文件名；
- 相对路径片段；
- 标签显示名；
- 来源显示名。

搜索完全本地执行，不在日志记录查询词或逐项路径。Filter Chips 支持：

- 来源；
- 标签；
- 可用状态；
- 文件格式；
- 有无标签。

选择多个标签时默认“全部满足”（ALL/AND），因为用户通常在逐步缩小结果；界面在第二个标签出现时明确显示 `全部标签`，并允许切换为 `任一标签`（ANY/OR）。不能隐藏布尔语义。

标签状态筛选必须绑定具体标签，例如“家人：已拒绝”，不能用模糊的全局“已拒绝”让用户猜测是对哪个标签的决定。

阶段 1 提供系统入口：

- 无标签；
- 来源不可用；
- 文件缺失；
- 文件不可读；
- 格式不支持。

不实现完整 `AND / OR / NOT` 查询构建器、不保存 Smart Collection，也不加入模型置信度条件。

### UI-008：稳定时间排序 — Approved

“全部照片”默认按可用媒体创建时间倒序；没有可靠媒体时间时回退到文件修改时间，最终以不可变 `asset_id` 保证稳定次序。阶段 1 允许切换：

- 最新优先；
- 最早优先；
- 文件名升序。

网格不按日期创建可折叠分组，避免在首个切片同时引入复杂 section pagination。当前排序必须对范围选择和下一页游标可复现。

## 9. 活动与错误状态

### UI-009：活动是一级可见能力，不是完整独立工作区 — Approved

阶段 1 同时提供：

- Toolbar 活动按钮：有任务时显示聚合状态和问题数量；
- 活动弹层：快速查看当前任务和控制；
- Sidebar“活动”页面：查看每个来源的当前/最近任务与错误摘要；
- 有任务时的底部上下文状态栏。

活动页面显示：

- 当前阶段，例如“扫描文件”“读取元数据”“生成缩略图”；
- 已处理数量、发现数量、错误数量和运行时间；
- 只有总量已知时才显示百分比与剩余数量；
- pause、resume、cancel 与 retry 的真实可用状态；
- 来源离线、等待重新授权和磁盘安全余量不足；
- 聚合错误码与受影响数量，不显示逐项完整路径。

阶段 1 不承诺 CPU/GPU 限制、仅接通电源时运行、详细执行历史、日志导出或跨 App 退出继续运行。这些属于阶段 4 调度与发布体验。

### UI-010：状态不伪装成空白或删除 — Approved

| 状态 | 网格呈现 | 主要动作 |
|---|---|---|
| 缩略图排队/生成中 | 中性占位与轻量进度，不改变布局 | 后台自动完成 |
| 单资产不可读 | 保留资产单元与错误徽标 | 查看摘要、稍后重试 |
| 格式不支持 | 通用文件图标与格式说明 | 查看元数据 |
| 文件缺失 | 保留历史单元与人工标签 | 查看来源、重扫 |
| 来源离线 | 页面顶部非模态横幅，保留旧索引 | 重新连接磁盘或重试 |
| 需要重新授权 | 明确权限横幅 | 通过系统选择器重新授权 |
| 缓存空间不足 | 停止新派生数据落盘；当前网格与 Inspector 可用内存临时图继续浏览 | 释放磁盘空间；缓存设置延期 |
| 查询无结果 | 显示当前筛选摘要 | 清除部分或全部筛选 |

来源离线、文件缺失或缩略图失败都不能把照片从用户当前心智模型中静默消失，更不能清除人工标签。

## 10. 视觉与可访问性基线

### UI-011：macOS 原生中性视觉 — Approved

- 使用系统字体、SF Symbols、Toolbar、Sidebar selection、Sheet、Context Menu 和标准焦点环；
- 支持 Light/Dark Mode，不自造一套与系统脱节的窗口 chrome；
- 中央照片区域使用中性背景，选中采用系统 Accent Color；
- 人工确认、明确错误等状态只用克制的小标识，不给每个标签铺大面积颜色；
- 颜色不是状态的唯一载体，徽标必须同时有形状、文字或辅助标签；
- 所有高频动作可由键盘到达，焦点顺序稳定；
- 缩略图、按钮和状态具有 VoiceOver 可理解名称；
- Reduce Motion 开启时不使用大幅网格重排动画；
- 窗口缩放和面板折叠不能丢失选择或任务状态。

阶段 1 不定义独立品牌色系统或自绘控件库。产品视觉辨识度先来自照片工作流、间距、信息层级和交互速度。

## 11. 后端契约影响

阶段 1 后端实施规格必须落实：

1. 统一图库、单来源、单标签、系统状态和组合筛选的 keyset 查询端口；
2. 稳定时间回退排序和 `asset_id` tie-break；
3. 网格轻量 projection 与 Inspector 按需详情 projection；
4. 多选标签三态聚合与原子批量决定端口；
5. 搜索与标签 ALL/ANY 的精确数据库语义；
6. 活动中心需要的 Job/Source 聚合查询，不暴露 GRDB 类型；
7. Grid small、Grid regular、Preview 三类缓存表示及其版本；
8. 筛选、排序和 generation missing 查询所需的追加索引；
9. UI 只能消费 Application 层状态，不在 View 内读取文件系统、bookmark 或数据库；
10. 任何批量标签操作失败时整体回滚，并且只有成功事务进入 Undo 历史。

不得为了以后 Review、Smart Collection 或 Compare 预建 prediction、saved query、similarity group 或 automation rule schema。

## 12. 阶段 1 产品验收旅程

### 12.1 空库到可浏览

1. 用户看到只读说明；
2. 点击“连接照片文件夹”；
3. 通过系统选择器选择合成 fixture；
4. 立即进入图库并看到真实扫描状态；
5. 已处理照片渐进出现；
6. App 重启后恢复来源和任务；
7. 全程没有源端写入。

### 12.2 浏览与人工标签

1. 用户调整网格密度并连续滚动；
2. 用标准 macOS 语义单选、范围选择和多选；
3. 在 Inspector 新建一个标签并批量确认；
4. 对另一个标签明确拒绝；
5. 清除决定后回到未判断；
6. Undo 恢复上一批成功操作；
7. 重启后决定完整保留。

### 12.3 搜索与筛选

1. 用文件名或相对路径片段搜索；
2. 添加来源与多个标签 chip；
3. 验证默认 ALL 与切换 ANY 的结果；
4. 进入无标签、缺失、不可读和不支持入口；
5. 大结果集滚动时排序稳定、没有重复或跳项。

### 12.4 故障恢复

1. 使用可丢弃 fixture 模拟来源离线、权限失效、损坏图片和低缓存空间；
2. UI 保留资产、人工标签和清晰状态；
3. 用户能重试或重新授权；
4. 不完整 generation 不产生批量 missing；
5. 活动中心不泄漏完整路径或图片内容。

自动化验收只使用测试创建的临时 fixture。受保护真实年份目录的只读人工 smoke 必须另行满足
[`LOCAL-TEST-DATA-SAFETY.md`](./LOCAL-TEST-DATA-SAFETY.md) 的授权门；`.photoslibrary` 在阶段 1 仍禁止读取或遍历。

## 14. 参考方向

- [Eagle](https://eagle.cool/)：Sidebar、网格、Inspector 与标签组织；

- HoudahSpot：未来从 Filter Chips 演进到高级布尔查询；
- Hazel：未来规则必须有 Dry Run、范围预览和撤销；
- Things：渐进披露与克制视觉；
- Apple 平台规范：[Sidebar](https://developer.apple.com/design/human-interface-guidelines/sidebars)、Toolbar、Inspector、[进度](https://developer.apple.com/design/human-interface-guidelines/progress-indicators)和辅助功能语义。
